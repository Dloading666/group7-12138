# 流程管理动态配置功能使用指南

## 🚀 快速开始

### 1. 执行数据库脚本

```bash
# 在 MySQL 中执行
mysql -u root -p management_system < backend/src/main/resources/db/workflow_schema.sql
```

### 2. 启动后端服务

```bash
cd backend
mvn spring-boot:run
```

### 3. 前端配置

在前端项目创建 `frontend/src/api/workflow.js` 文件（已完成）

## 📋 API 接口说明

### 节点类型接口

#### 1. 获取所有启用的节点类型
```
GET /api/workflows/node-types
Authorization: Bearer {token}

响应示例：
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "type": "start",
      "name": "开始节点",
      "icon": "VideoPlay",
      "color": "#52c41a",
      "category": "基础节点",
      "sortOrder": 1,
      "enabled": true
    }
  ]
}
```

#### 2. 创建节点类型
```
POST /api/workflows/node-types
Authorization: Bearer {token}
Content-Type: application/json

{
  "type": "custom-task",
  "name": "自定义任务节点",
  "icon": "Setting",
  "color": "#1890ff",
  "category": "自定义节点",
  "sortOrder": 11,
  "enabled": true,
  "description": "自定义的任务节点"
}
```

### 流程接口

#### 3. 创建流程
```
POST /api/workflows
Authorization: Bearer {token}
Content-Type: application/json

{
  "name": "商品价格采集流程",
  "description": "采集京东和淘宝的商品价格数据"
}
```

#### 4. 更新流程配置
```
PUT /api/workflows/1
Authorization: Bearer {token}
Content-Type: application/json

{
  "name": "商品价格采集流程",
  "description": "更新后的描述",
  "config": "{\"nodes\":[...]}"
}
```

#### 5. 发布流程
```
POST /api/workflows/1/publish
Authorization: Bearer {token}
```

## ✨ 核心优势

### 之前（写死在前端）
```javascript
// ❌ 硬编码，修改需要改代码
const nodeTypes = [
  { type: 'start', label: '开始', icon: 'VideoPlay' }
```

## 🎯 测试步骤

```bash

  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

2. **获取节点类型**
```bash
curl -X GET http://localhost:8080/api/workflows/node-types \
  -H "Authorization: Bearer {token}"
```

3. **创建流程**
```bash
curl -X POST http://localhost:8080/api/workflows \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"name":"测试流程","description":"测试描述"}'
```

4. **发布流程**
```bash
curl -X POST http://localhost:8080/api/workflows/1/publish \
  -H "Authorization: Bearer {token}"
```

## 💡 注意事项

1. **权限控制**
   - 只有管理员可以创建/编辑节点类型
   - 普通用户只能查看启用的节点类型

2. **数据一致性**
   - 删除流程时会级联删除所有节点
   - 禁用节点类型不影响已创建的流程

3. **性能优化**
   - 节点类型数据建议缓存
   - 流程配置可以存储为 JSON 减少表关联查询
