# 指标计算参数位置和操作说明

本文档详细说明计算经营规模指标所需的所有参数在页面中的具体位置，以及如何通过爬虫获取这些数据。

## 📋 指标计算所需参数总览

根据您提供的指标计算逻辑，需要以下参数：

### 入参参数（3个）
1. **taxNo** - 纳税人识别号
2. **uscCode** - 统一社会信用代码  
3. **appDate** - 申请日期

### 发票信息参数（4个字段，每条发票记录）
1. **sign** - 发票类型（销项/进项）
2. **state** - 发票状态（正常/异常）
3. **invoice_time** - 发票开票时间
4. **jshj** - 价税合计

---

## 🎯 第一步：在企业信息查询页获取企业身份参数（taxNo、uscCode）

### 页面位置：企业信息查询页面 (`/enterprise-info`)

#### 操作流程：

1. **访问企业信息查询页面**
   - URL路径：`http://localhost:3000/enterprise-info`
   - 或点击导航栏的"企业信息查询"链接

2. **输入企业名称并查询**
   
   **位置1：企业名称输入框**
   - 页面位置：查询区第一个输入框
   - 标签文字："企业名称"
   - **元素ID：`enterprise-name-input`**
   - 操作：爬虫输入企业名称（用于触发查询）
   - XPath示例：`//input[@id='enterprise-name-input']`
   - CSS选择器：`#enterprise-name-input`

   **位置2：查询按钮**
   - 按钮文字："查询"
   - 操作：爬虫点击按钮触发查询结果展示

3. **从“企业基本信息”结果区提取企业身份参数**
   
   **位置3：纳税人识别号（taxNo）**
   - 页面位置：结果区信息项（标签文字："纳税人识别号："）
   - **元素ID：`tax-no`**
   - 操作：爬虫读取该元素的文本内容作为 `taxNo`
   - XPath示例：`//span[@id='tax-no']`
   - CSS选择器：`#tax-no`
   
   **位置4：统一社会信用代码（uscCode）**
   - 页面位置：结果区信息项（标签文字："统一社会信用代码："）
   - **元素ID：`usc-code`**
   - 操作：爬虫读取该元素的文本内容作为 `uscCode`
   - XPath示例：`//span[@id='usc-code']`
   - CSS选择器：`#usc-code`

#### 💡 爬虫建议：
- **推荐方式**：在企业信息查询页先“查到并读取” `taxNo/uscCode`，用于后续页面的表单填充与发票查询（更符合“企业身份先确定”的业务逻辑）

---

## 🎯 第二步：在业务申请页获取申请日期（appDate），并用 taxNo/uscCode 填充申请表单

### 页面位置：业务申请页面 (`/application`)

#### 操作流程：

1. **访问业务申请页面**
   - URL路径：`http://localhost:3000/application`
   - 或点击导航栏的"业务申请"链接

2. **用第一步拿到的企业身份参数填充表单**
   
   **位置5：纳税人识别号输入框**
   - 页面位置：表单第一个输入框
   - 标签文字："纳税人识别号 *"（必填）
   - **元素ID：`app-tax-no`**
   - 操作：爬虫将第一步的 `taxNo` 填入该输入框
   - XPath示例：`//input[@id='app-tax-no']`
   - CSS选择器：`#app-tax-no`
   
   **位置6：统一社会信用代码输入框**
   - 页面位置：表单第二个输入框
   - 标签文字："统一社会信用代码 *"（必填）
   - **元素ID：`app-usc-code`**
   - 操作：爬虫将第一步的 `uscCode` 填入该输入框
   - XPath示例：`//input[@id='app-usc-code']`
   - CSS选择器：`#app-usc-code`
   
3. **填写并获取申请日期（appDate）**
   
   **位置7：申请日期选择器**
   - 页面位置：表单第三个输入框
   - 标签文字："申请日期 *"
   - **元素ID：`app-date`**
   - 操作：爬虫选择/填写申请日期，并读取其值作为 `appDate`
   - 格式：YYYY-MM-DD（例如：2024-02-06）
   - XPath示例：`//input[@id='app-date']`
   - CSS选择器：`#app-date`

4. **（可选）提交申请并从结果区再次确认入参**
   
   提交后会出现成功提示框，可用于校验最终入参是否与填写一致：
   - `app-tax-no-display` → taxNo
   - `app-usc-code-display` → uscCode
   - `app-date-display` → appDate

## 🎯 第三步：在发票查询页查询并提取发票信息（用于计算）

### 页面位置：发票查询页面 (`/invoice-query`)

#### 操作流程：

1. **访问发票查询页面**
   - URL路径：`http://localhost:3000/invoice-query`
   - 或点击导航栏的"发票查询"链接

2. **填写查询条件（taxNo / uscCode）**
   
   **位置8：纳税人识别号查询输入框**
   - **元素ID：`invoice-tax-no`**
   - 操作：填写第二步确认的 `taxNo`
   
   **位置9：统一社会信用代码查询输入框**
   - **元素ID：`invoice-usc-code`**
   - 操作：填写第二步确认的 `uscCode`

3. **点击查询按钮并等待结果表格出现**

### 页面位置：发票查询页面的结果表格

查询成功后，页面会显示一个表格，包含所有发票记录。每条发票记录包含以下字段：

#### 表格结构：

| 序号 | 发票代码 | 发票号码 | 发票类型 | 发票状态 | 开票时间 | 价税合计 | 操作 |
|------|---------|---------|---------|---------|---------|---------|------|
| 1    | ...     | ...     | 销项     | 正常     | ...     | ...     | ...  |
| 2    | ...     | ...     | 销项     | 正常     | ...     | ...     | ...  |

#### 关键字段提取：

**位置11：发票类型（sign）**
- 页面位置：表格第4列（"发票类型"列）
- 显示内容：绿色标签显示"销项"，橙色标签显示"进项"
- **元素ID：`invoice-sign-{index}`**（index从0开始，对应表格行号-1）
  - 第1行：`invoice-sign-0`
  - 第2行：`invoice-sign-1`
  - 第3行：`invoice-sign-2`
  - ...以此类推
- 操作：爬虫读取这个元素的文本内容
- 可能的值：`"销项"` 或 `"进项"`
- XPath示例：`//span[@id='invoice-sign-0']`
- CSS选择器：`#invoice-sign-0`、`#invoice-sign-1` 等
- **筛选条件**：只保留 `sign = '销项'` 的记录

**位置12：发票状态（state）**
- 页面位置：表格第5列（"发票状态"列）
- 显示内容：蓝色标签显示"正常"，红色标签显示"异常"
- **元素ID：`invoice-state-{index}`**（index从0开始）
  - 第1行：`invoice-state-0`
  - 第2行：`invoice-state-1`
  - 第3行：`invoice-state-2`
  - ...以此类推
- 操作：爬虫读取这个元素的文本内容
- 可能的值：`"正常"` 或 `"异常"`
- XPath示例：`//span[@id='invoice-state-0']`
- CSS选择器：`#invoice-state-0`、`#invoice-state-1` 等
- **筛选条件**：只保留 `state = '正常'` 的记录

**位置13：发票开票时间（invoice_time）**
- 页面位置：表格第6列（"开票时间"列）
- 显示格式：`YYYY-MM-DD HH:mm:ss`（例如：`2024-01-15 10:30:00`）
- **元素ID：`invoice-time-{index}`**（index从0开始）
  - 第1行：`invoice-time-0`
  - 第2行：`invoice-time-1`
  - 第3行：`invoice-time-2`
  - ...以此类推
- 操作：爬虫读取这个元素的文本内容
- XPath示例：`//td[@id='invoice-time-0']`
- CSS选择器：`#invoice-time-0`、`#invoice-time-1` 等
- **筛选条件**：发票开票时间与申请日期（appDate）的月份差在 1-12 个月之间（不含当月）
  - 例如：如果 appDate = 2024-02-06
  - 则只保留 invoice_time 在 2023-02-06 到 2024-01-06 之间的发票

**位置14：价税合计（jshj）**
- 页面位置：表格第7列（"价税合计"列）
- 显示格式：带千分位分隔符的数字（例如：`123,456.79`）
- **元素ID：`invoice-jshj-{index}`**（index从0开始）
  - 第1行：`invoice-jshj-0`
  - 第2行：`invoice-jshj-1`
  - 第3行：`invoice-jshj-2`
  - ...以此类推
- 操作：爬虫读取这个元素的文本内容
- XPath示例：`//td[@id='invoice-jshj-0']`
- CSS选择器：`#invoice-jshj-0`、`#invoice-jshj-1` 等
- **数据处理**：
  1. 去除千分位分隔符（逗号）
  2. 转换为数字类型
  3. 对筛选后的所有发票的 jshj 求和

#### 💡 爬虫提取发票数据的建议：

1. **遍历表格行**
   ```python
   # 伪代码示例
   rows = driver.find_elements(By.XPATH, "//tbody/tr")
   for index, row in enumerate(rows):
       sign = row.find_element(By.ID, f"invoice-sign-{index}").text
       state = row.find_element(By.ID, f"invoice-state-{index}").text
       invoice_time = row.find_element(By.ID, f"invoice-time-{index}").text
       jshj_text = row.find_element(By.ID, f"invoice-jshj-{index}").text
       jshj = float(jshj_text.replace(',', ''))  # 去除千分位分隔符
   ```

2. **数据筛选逻辑**
   ```python
   # 伪代码示例
   filtered_invoices = []
   for invoice in invoices:
       # 筛选条件1：sign = '销项'
       if invoice['sign'] != '销项':
           continue
       # 筛选条件2：state = '正常'
       if invoice['state'] != '正常':
           continue
       # 筛选条件3：月份差在1-12个月之间（不含当月）
       if is_in_date_range(invoice['invoice_time'], appDate):
           filtered_invoices.append(invoice)
   ```

3. **计算求和**
   ```python
   # 伪代码示例
   sale_jshj_sum = sum(invoice['jshj'] for invoice in filtered_invoices)
   if sale_jshj_sum is None or sale_jshj_sum == 0:
       result = 0
   else:
       result = round(sale_jshj_sum, 5)  # 四舍五入保留5位小数
   ```

## 📊 完整爬虫操作流程总结（推荐顺序）

### 步骤1：企业信息查询页拿 taxNo / uscCode
1. 访问 `/enterprise-info` 页面
2. 输入企业名称 `enterprise-name-input` 并点击查询
3. 从结果区读取：
   - `tax-no` → taxNo
   - `usc-code` → uscCode

### 步骤2：业务申请页确定 appDate（并填充 taxNo/uscCode）
1. 访问 `/application` 页面
2. 填写：
   - `app-tax-no` = taxNo
   - `app-usc-code` = uscCode
   - `app-date` = appDate
3. （可选）提交后从结果区校验：
   - `app-tax-no-display`、`app-usc-code-display`、`app-date-display`

### 步骤3：发票查询页查询并提取发票数据
1. 访问 `/invoice-query` 页面
2. 填写查询条件：
   - `invoice-tax-no` = taxNo
   - `invoice-usc-code` = uscCode
3. 点击查询按钮，等待结果表格出现
1. 遍历表格每一行（从 index=0 开始）
2. 提取每条发票的：
   - `invoice-sign-{index}` → sign
   - `invoice-state-{index}` → state
   - `invoice-time-{index}` → invoice_time
   - `invoice-jshj-{index}` → jshj（需去除千分位分隔符）

### 步骤4：数据筛选和计算
1. 筛选条件：
   - sign = '销项'
   - state = '正常'
   - invoice_time 与 appDate 的月份差在 1-12 个月之间（不含当月）
2. 对筛选后的 jshj 求和
3. 如果结果为空，返回 0；否则四舍五入保留5位小数

## ⚠️ 注意事项

1. **日期格式处理**
   - appDate 格式：`YYYY-MM-DD`
   - invoice_time 格式：`YYYY-MM-DD HH:mm:ss`
   - 月份差计算需要排除当月

2. **金额格式处理**
   - 页面显示的金额可能包含千分位分隔符（逗号）
   - 爬取后需要去除逗号再转换为数字

3. **表格行号**
   - 页面显示的序号从1开始
   - 但元素ID中的index从0开始
   - 第1行对应 index=0，第2行对应 index=1，以此类推

4. **动态内容等待**
   - 提交表单后需要等待成功提示框出现
   - 点击查询按钮后需要等待表格数据加载完成

5. **数据验证**
   - 建议使用页面底部的"销项发票合计"（`total-sale-amount`）进行数据验证
   - 但注意：这个合计包含所有销项发票，不包含筛选条件（状态、时间范围）
