# 指标2 参数位置和操作说明

**指标编码**：`inv_f1_12m_down_sale_month_cv_teach`  
**指标含义**：近1-12个月（不含当月）销项发票月度金额波动系数，用于衡量经营稳定性。

本页数据与**经营规模指标（指标1）**的数据页面分离，所有参数均在 **经营稳定性指标** 独立页面中提供。

---

## 一、指标计算所需参数总览

### 1. 入参（3 个）

| 参数名   | 含义               | 用途说明 |
|----------|--------------------|----------|
| `taxNo`  | 纳税人识别号       | 与 uscCode 一起定位企业发票数据 |
| `uscCode`| 统一社会信用代码   | 同上 |
| `appDate`| 申请日期           | 用于判断「近1-12个月不含当月」的时间范围 |

### 2. 发票明细（每条记录 4 个字段）

| 参数名        | 含义         | 用途说明 |
|---------------|--------------|----------|
| `sign`        | 发票类型     | 需筛选为「销项」 |
| `state`       | 发票状态     | 需筛选为「正常」 |
| `invoice_time`| 开票时间     | 用于按月份分组、以及判断是否在 1–12 个月（不含当月） |
| `jshj`        | 价税合计     | 用于按月份汇总 `month_sale_amt`，再算均值、标准差、波动系数 |

计算逻辑简述：先按 `taxNo`、`uscCode` 取企业发票，筛出销项+正常+时间在 appDate 前 1–12 个月（不含当月）的记录；按开票月份 YYYY-MM 分组，对 `jshj` 求和得到各月 `month_sale_amt`；再算 `mean_amt`、`std_amt`、`cv = std_amt / mean_amt`，得到指标结果。

---

## 二、数据所在页面（与指标1 分离）

- **页面名称**：经营稳定性指标（近1-12个月销项发票月度金额波动系数）
- **访问路径**：`/stability-indicator`
- **完整 URL 示例**：`http://localhost:3000/stability-indicator`

本页专门用于指标2，与「企业信息查询」「业务申请」「发票查询」等指标1 相关页面分开，避免爬虫混用。

---

## 三、参数在页面上的具体位置与操作步骤

### 第一步：打开经营稳定性指标页并获取入参

1. **打开页面**  
   访问：`/stability-indicator`（或从首页点击「经营稳定性指标」）。

2. **获取纳税人识别号（taxNo）与统一社会信用代码（uscCode）**  
   - 在「一、入参」区域找到 **企业名称** 输入框。  
   - **元素 ID**：`stability-enterprise-name-input`  
   - **操作**：输入企业名称后，点击 **「查询企业」** 按钮。  
   - 下方会展示：  
     - **纳税人识别号** 对应元素 ID：`stability-tax-no`（爬虫取此处文本即 `taxNo`）。  
     - **统一社会信用代码** 对应元素 ID：`stability-usc-code`（爬虫取此处文本即 `uscCode`）。

3. **获取申请日期（appDate）**  
   - 在同一区域有 **「申请日期（用于计算近1-12个月不含当月）」** 的日期选择框。  
   - **元素 ID**：`stability-app-date`  
   - **操作**：爬虫可在此选择或读取日期，格式为 `YYYY-MM-DD`，该值即 `appDate`。

以上三步完成后，入参 `taxNo`、`uscCode`、`appDate` 即可从本页一次性拿到（无需再跳转企业信息页或业务申请页）。

---

### 第二步：查询发票并获取每条记录的 sign、state、invoice_time、jshj

1. **填写发票查询条件**  
   在「二、发票明细」区域：  
   - **纳税人识别号** 输入框：ID `stability-invoice-tax-no`（可填第一步得到的 `taxNo`）。  
   - **统一社会信用代码** 输入框：ID `stability-invoice-usc-code`（可填第一步得到的 `uscCode`）。  
   - 开票日期起、止为可选，ID 分别为：`stability-invoice-start-date`、`stability-invoice-end-date`。

2. **点击「查询发票」**  
   点击后，下方会出现发票列表表格。

3. **从表格中逐行抓取每条发票的 4 个字段**  
   表格每行对应一条发票，字段与元素 ID 对应关系如下（`index` 为行号，从 0 开始）：

   | 含义     | 元素 ID | 说明 |
   |----------|---------|------|
   | 发票类型 sign | `stability-invoice-sign-{index}`   | 文本为「销项」或「进项」，指标仅保留「销项」。 |
   | 发票状态 state| `stability-invoice-state-{index}`  | 文本为「正常」或「异常」，指标仅保留「正常」。 |
   | 开票时间 invoice_time | `stability-invoice-time-{index}` | 格式 `YYYY-MM-DD HH:mm:ss`，用于月份分组与时间范围筛选。 |
   | 价税合计 jshj | `stability-invoice-jshj-{index}`   | 页面上可能带千分位，爬取后需去掉逗号再参与计算。 |

   例如：  
   - 第 1 行：`stability-invoice-sign-0`、`stability-invoice-state-0`、`stability-invoice-time-0`、`stability-invoice-jshj-0`  
   - 第 2 行：`stability-invoice-sign-1`、…  
   依此类推。

---

## 四、爬虫操作流程建议（仅用本页即可完成指标2 数据采集）

1. 访问 `/stability-indicator`。
2. **入参**  
   - 在 `stability-enterprise-name-input` 输入企业名称，点击「查询企业」。  
   - 从 `stability-tax-no`、`stability-usc-code` 读取 `taxNo`、`uscCode`。  
   - 从 `stability-app-date` 读取或设置 `appDate`。
3. **发票明细**  
   - 将 `taxNo`、`uscCode` 填入 `stability-invoice-tax-no`、`stability-invoice-usc-code`（若已自动带出可不再填）。  
   - 点击「查询发票」。  
   - 等待表格出现后，遍历每一行，按 `index` 读取：  
     - `stability-invoice-sign-{index}` → sign  
     - `stability-invoice-state-{index}` → state  
     - `stability-invoice-time-{index}` → invoice_time  
     - `stability-invoice-jshj-{index}` → jshj（去掉千分位逗号后转为数字）。
4. **计算**  
   - 筛出 `sign='销项'`、`state='正常'`，且 `invoice_time` 与 `appDate` 的月份差在 1–12 个月（不含当月）。  
   - 按 `invoice_time` 所在月份 YYYY-MM 分组，对 `jshj` 求和得各月 `month_sale_amt`。  
   - 计算 `mean_amt`、`std_amt`、`cv = std_amt / mean_amt`；若 `mean_amt` 为空或为 0，则 `cv = 999`。  
   - 输出：`{ "mean_amt": ..., "std_amt": ..., "cv": ... }`（数值四舍五入保留 5 位小数）。

## 五、元素 ID 速查（仅指标2 页面）

| 用途 | 元素 ID |
|------|---------|
| 企业名称输入 | `stability-enterprise-name-input` |
| 纳税人识别号（入参） | `stability-tax-no` |
| 统一社会信用代码（入参） | `stability-usc-code` |
| 申请日期 | `stability-app-date` |
| 发票查询-纳税人识别号 | `stability-invoice-tax-no` |
| 发票查询-统一社会信用代码 | `stability-invoice-usc-code` |
| 发票查询-开票日期起 | `stability-invoice-start-date` |
| 发票查询-开票日期止 | `stability-invoice-end-date` |
| 第 i 行-发票类型 | `stability-invoice-sign-{i}` |
| 第 i 行-发票状态 | `stability-invoice-state-{i}` |
| 第 i 行-开票时间 | `stability-invoice-time-{i}` |
| 第 i 行-价税合计 | `stability-invoice-jshj-{i}` |

其中 `{i}` 为从 0 开始的表格行下标。

## 六、与指标1 数据页的区分

- **指标1（经营规模）**：使用「企业信息查询」「业务申请」「发票查询」等页面，入参及发票相关元素 ID 为 `tax-no`、`usc-code`、`app-date`、`invoice-sign-{i}` 等（无 `stability-` 前缀）。  
- **指标2（经营稳定性）**：仅使用「经营稳定性指标」页 `/stability-indicator`，所有相关元素 ID 均带 `stability-` 前缀，便于爬虫只抓本页即可算指标2，且与指标1 数据源在页面上完全分离。
