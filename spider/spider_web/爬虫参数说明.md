# 爬虫参数说明文档

本文档列出了计算经营规模指标所需的所有参数及其在网页中的位置。

## 指标计算所需参数

### 1. 入参参数

#### 1.1 纳税人识别号 (taxNo)
- **用途**: 用于查询企业发票信息
- **页面**: 企业信息查询页面、发票查询页面、申请页面
- **元素ID**: 
  - 企业信息页面: `tax-no`
  - 发票查询页面: `invoice-tax-no`
  - 申请页面: `app-tax-no`
  - 申请成功页面: `app-tax-no-display`

#### 1.2 统一社会信用代码 (uscCode)
- **用途**: 用于查询企业发票信息
- **页面**: 企业信息查询页面、发票查询页面、申请页面
- **元素ID**:
  - 企业信息页面: `usc-code`
  - 发票查询页面: `invoice-usc-code`
  - 申请页面: `app-usc-code`
  - 申请成功页面: `app-usc-code-display`

#### 1.3 申请日期 (appDate)
- **用途**: 用于计算发票时间范围（近1-12个月，不含当月）
- **页面**: 申请页面
- **元素ID**:
  - 申请页面输入框: `app-date`
  - 申请成功页面显示: `app-date-display`

### 2. 发票信息参数

发票信息在**发票查询页面** (`/invoice-query`) 的表格中展示。

#### 2.1 发票类型 (sign)
- **用途**: 筛选销项发票（sign = '销项'）
- **页面**: 发票查询页面
- **元素ID**: `invoice-sign-{index}` (index为行号，从0开始)
- **可能的值**: '销项' 或 '进项'
- **样式类**: 
  - 销项: `status-sale` (绿色背景)
  - 进项: `status-purchase` (橙色背景)

#### 2.2 发票状态 (state)
- **用途**: 筛选正常发票（state = '正常'）
- **页面**: 发票查询页面
- **元素ID**: `invoice-state-{index}` (index为行号，从0开始)
- **可能的值**: '正常' 或 '异常'
- **样式类**:
  - 正常: `status-normal` (蓝色背景)
  - 异常: `status-abnormal` (红色背景)

#### 2.3 发票开票时间 (invoice_time)
- **用途**: 计算与申请日期的月份差（需在1-12个月之间，不含当月）
- **页面**: 发票查询页面
- **元素ID**: `invoice-time-{index}` (index为行号，从0开始)
- **格式**: 'YYYY-MM-DD HH:mm:ss' (例如: '2024-01-15 10:30:00')

#### 2.4 价税合计 (jshj)
- **用途**: 对符合条件的发票进行求和
- **页面**: 发票查询页面
- **元素ID**: `invoice-jshj-{index}` (index为行号，从0开始)
- **格式**: 数字，可能包含千分位分隔符（例如: 123,456.79）

## 页面访问路径

1. **首页**: `/`
2. **登录页**: `/login`
3. **企业信息查询**: `/enterprise-info`
4. **发票查询**: `/invoice-query`
5. **业务申请**: `/application`

## 爬虫爬取流程建议

### 步骤1: 获取入参
1. 访问申请页面 (`/application`)
2. 填写表单并提交，获取申请编号
3. 从申请成功页面提取：
   - `app-tax-no-display` → taxNo
   - `app-usc-code-display` → uscCode
   - `app-date-display` → appDate

### 步骤2: 查询发票信息
1. 访问发票查询页面 (`/invoice-query`)
2. 使用获取的 taxNo 和 uscCode 填写查询表单
3. 点击查询按钮
4. 遍历表格中的每一行发票记录：
   - 提取 `invoice-sign-{index}` → sign
   - 提取 `invoice-state-{index}` → state
   - 提取 `invoice-time-{index}` → invoice_time
   - 提取 `invoice-jshj-{index}` → jshj（需要去除千分位分隔符）

### 步骤3: 数据筛选和计算
1. 筛选条件：
   - sign = '销项'
   - state = '正常'
   - invoice_time 与 appDate 的月份差在 1-12 个月之间（不含当月）
2. 对筛选后的 jshj 字段求和
3. 如果结果为空，返回 0；否则四舍五入保留5位小数

## 其他有用的元素ID

- `invoice-code-{index}`: 发票代码
- `invoice-number-{index}`: 发票号码
- `total-sale-amount`: 销项发票合计金额（可用于验证）
- `total-purchase-amount`: 进项发票合计金额

## 注意事项

1. 所有金额字段在页面中可能包含千分位分隔符，爬取后需要去除
2. 日期格式需要转换为标准格式进行计算
3. 月份差计算需要排除当月（例如：如果申请日期是2024-02-06，则只计算2024-01-06之前到2023-02-06之间的发票）
4. 表格行号从0开始，但页面显示从1开始
